HOJA DE REQUISITOS

Id CU2
Nombre Registro
Actores Usuarios
Descripción El usuario quiere registrarse en la aplicacion
Precondiciones
  • El usuario debe de estar en la página de registro de usuario
  • El usuario si tiene una cuenta en la aplicacion no debee de tener ninguna sesion abierta, porque le redirigirá adentro de ella
Secuencia normal
  1. El usuario se encuentra en la página
  2. El usuario introduce las credenciales que se piden:
    • Usuario
    • Contraseña (2 veces para evitar problemas)
    • Correo electónico
  3. El usuario le da a botón de registrarse
Postcondiciones
  • El usuario es registrado correctamente en la aplicación
  • Se redirigirá al inicio de la aplicación
Secuencia alternativa
  • No puede registrar al usuario porque puede haber un fallo en la Base de Datos o en el servidor
  • No puede registrar al usuario porque ya hay uno con:
    • El mismo nombre de usuario (es únuico)
    • El mismo correo electrónico (es único)
